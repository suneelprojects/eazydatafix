# Easy Data Fix - User Personas

# Primary Users

## 1. Data Analyst

### Goal

Prepare datasets quickly for reporting and business intelligence.

### Challenges

* Missing values
* Duplicate records
* Incorrect data types
* Inconsistent formats

### Success Criteria

Upload a dataset, receive recommendations, and produce a clean dataset within minutes.

---

## 2. Data Scientist

### Goal

Prepare high-quality datasets for machine learning.

### Challenges

* Feature consistency
* Data validation
* Outlier detection
* Missing values
* Data leakage

### Success Criteria

Receive a validated dataset ready for model training.

---

## 3. Student

### Goal

Learn data preprocessing and understand data quality.

### Challenges

* Limited experience
* Doesn't know which cleaning steps to perform

### Success Criteria

Easy-to-understand recommendations with explanations.

---

## 4. Trainer / Educator

### Goal

Teach data quality concepts using real datasets.

### Challenges

* Demonstrating data issues
* Explaining why fixes are needed

### Success Criteria

Framework explains every recommendation clearly.

---

## 5. Organization

### Goal

Improve enterprise data quality.

### Challenges

* Large datasets
* Repeated quality issues
* Compliance

### Success Criteria

Automated assessment, reporting, validation, and monitoring.

---

# Secondary Users

* Researchers
* Data Engineers
* Business Analysts
* AI Engineers
* Open Source Contributors

---

# Design Principle

Every new feature must benefit at least one defined user persona.

If a feature does not clearly help a user persona, it should not be added.
